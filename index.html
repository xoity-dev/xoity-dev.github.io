Paradigm: Greedy. Two smallest frequencies are merged at each step.
Algorithm Overview

Huffman coding is a lossless compression method. Frequent symbols get shorter bit codes, rare symbols get longer codes.

It uses a min-heap (priority queue): repeatedly extract two smallest nodes, merge them, and push the merged node back.

Common mistake: assigning codes before tree construction finishes. Codes are only valid after the final root is built.
Worked Example

Input: A:5, B:9, C:12, D:13, E:16, F:45.

  1. Merge A(5)+B(9)=14
  2. Merge C(12)+D(13)=25
  3. Merge 14+E(16)=30
  4. Merge 25+30=55
  5. Merge F(45)+55=100 (root)

Traverse left as 0 and right as 1 to get final codes.
