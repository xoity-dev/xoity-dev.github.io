NP-Completeness Proof (Hamiltonian Path -> Longest Path)

Decision problem: Given directed graph G, vertices s,t, and integer k, does there exist a simple path from s to t with at least k edges?

NP membership: Certificate is path P. Verifier checks in polynomial time: (1) first vertex is s, last is t, (2) each consecutive pair is an edge in G, (3) no repeated vertices, (4) |P|-1 >= k.

Reduction construction: From Hamiltonian Path instance (G,s,t), construct Longest Path instance (G,s,t,k=|V|-1). No graph modification needed beyond setting k.

Correctness (forward): If Hamiltonian path exists, it visits all |V| vertices once, so length is exactly |V|-1, hence Longest Path instance is YES.

Correctness (backward): If Longest Path instance is YES with length >= |V|-1, a simple path cannot exceed |V|-1, so it equals |V|-1 and visits all vertices once, giving a Hamiltonian path.

Polynomiality argument: Construction copies input graph and computes k=|V|-1 in O(|V|+|E|). Verifier is polynomial. Therefore Longest Path is NP-hard and in NP, thus NP-complete.

DAG contrast: For DAGs, longest path is solvable in polynomial time via topological DP; NP-completeness applies to general directed graphs with cycles.
